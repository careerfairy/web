# Vulnerability & Patch Management

## Purpose

Reduce risk by timely handling of vulnerabilities and patches.

## Controls in place

-  Weekly dependency update cycle; critical CVEs handled within 7 days
-  GitHub Dependabot/SCA alerts triaged; ignore only with justification
-  Runtime images and tools kept up to date
-  Third-party processor notices monitored

## Evidence & Tooling

-  Dependency PRs, CVE dashboards, release notes

## Review Cadence & Owner

-  Monthly check-in; Owner: Head of Engineering
